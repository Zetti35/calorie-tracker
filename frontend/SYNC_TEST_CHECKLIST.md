## Тест 1: Базовая синхронизация ✅

### Шаги:
1. [ ] Открой http://localhost:3000
2. [ ] Открой Console (F12)
3. [ ] Проверь логи инициализации:
   ```
   [useSyncStore] Initializing sync...
   [Migration] Checking if migration is needed...
   [Sync Init] Starting sync initialization...
   ```
4. [ ] Перейди в раздел "Дневник"
5. [ ] Добавь любой продукт (например, "Яблоко 100г")
6. [ ] Подожди 2 секунды
7. [ ] Проверь логи синхронизации:
   ```
   [useSyncStore] Triggering debounced sync...
   [syncToServer] Success: 2026-04-21T...
   ```
8. [ ] Проверь индикатор в правом верхнем углу:
   - ⟳ (синий, крутится) → синхронизация
   - ✓ (зеленый) → синхронизировано
   - Исчезает через 2 секунды

### Проверка в Supabase:
9. [ ] Открой Supabase Dashboard
10. [ ] Table Editor → `diary_data`
11. [ ] Найди свою запись (по `user_id`)
12. [ ] Проверь колонку `data` - должны быть твои данные
13. [ ] Проверь `updated_at` - должна быть текущая дата/время

### Результат:
- [ ] ✅ Данные синхронизированы
- [ ] ❌ Ошибка (см. Console для деталей)

## Troubleshooting 🔧

### Ошибка: "No authentication"
- Проверь, что залогинен в Telegram Mini App
- Проверь Console: `window.Telegram.WebApp.initData` должен быть не пустым

### Ошибка: "401 Unauthorized"
- Telegram initData невалидный или истек
- Перезагрузи страницу

### Ошибка: "404 User not found"
- Пользователь не существует в таблице `users`
- Залогинься через `/api/auth/login` сначала

### Индикатор не появляется
- Проверь Console на ошибки
- Проверь, что `SyncProvider` добавлен в layout
- Проверь, что `_sync` state существует в store

### Данные не синхронизируются
- Проверь Network tab: должны быть запросы к `/api/sync`
- Проверь Console на ошибки
- Проверь Supabase connection
- Проверь RLS policies
