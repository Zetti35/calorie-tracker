# Калорийный Трекер — Steering Document

## О проекте

Приложение "Калорийный Трекер" на Python + Streamlit.
Цель: помочь пользователям считать калории, БЖУ, следить за питанием и тренировками.

Первая версия — Streamlit (быстрый старт).
Вторая версия — миграция на React + FastAPI.

---

## Цели обучения

- Освоить Python с нуля: type hints, Pydantic, модульная структура
- Научиться работать с Git: коммит после каждого важного шага
- Понять fullstack-разработку: от идеи до рабочего приложения
- Использовать ИИ-агентов как инструмент разработки

## Правила разработки

### Python
- Всегда используй type hints: `def foo(x: int) -> str:`
- Используй Pydantic-модели для всех структур данных
- Один модуль = одна ответственность
- Комментарии на русском языке

### Git
- Коммит после каждого важного шага
- Формат сообщения: `feat: описание` / `fix: описание` / `chore: описание`
- Никогда не коммить `.env` (там секретные ключи!)

### Streamlit
- Все надписи, кнопки и сообщения — на русском языке
- Данные сессии хранить в `st.session_state`
- Валидация ввода через `st.error()` и `st.warning()`

## Формулы

### BMR (Mifflin-St Jeor)
- Мужчины: `10 * вес + 6.25 * рост - 5 * возраст + 5`
- Женщины: `10 * вес + 6.25 * рост - 5 * возраст - 161`

### TDEE = BMR × коэффициент активности
| Уровень          | Коэффициент |
|------------------|-------------|
| Сидячий          | 1.2         |
| Лёгкая активность| 1.375       |
| Умеренная        | 1.55        |
| Активный         | 1.725       |
| Очень активный   | 1.9         |

### Цели
- Похудение: TDEE - 500 ккал
- Поддержание: TDEE
- Набор массы: TDEE + 500 ккал

### Макронутриенты (от итоговых калорий)
- Белки: 30% → делим на 4 (ккал/г)
- Жиры: 25% → делим на 9 (ккал/г)
- Углеводы: 45% → делим на 4 (ккал/г)

## Переменные окружения

Скопируй `.env.example` в `.env` и заполни:
```
OPENAI_API_KEY=твой_ключ_здесь
```

Без ключа приложение работает с заглушками — это нормально для старта.
